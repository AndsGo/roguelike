# v1.3.0 Relic System Activation + Build-Defining Relics Design

> Date: 2026-03-08
> Status: Approved
> Goal: Activate all 35 existing dormant relics + add 8 new build-defining relics

## Background

The relic system has complete data (35 relics in `relics.json` with `triggerEvent` and `effect` fields) but **no runtime evaluation**. Relics are acquired via events and stored in `RunState.relics`, but their effects are never applied during battle. This design adds the evaluation engine and 8 new "rule-changing" relics.

## Architecture: RelicSystem (Mixed Integration)

### Three Integration Layers

1. **Passive/Stat relics** — Applied in `Unit.getEffectiveStats()` pipeline (alongside synergy bonuses). Examples: +10% attack, +15% magic power.

2. **Reactive relics** — EventBus listeners registered by `RelicSystem` at battle start. Matches held relics' `triggerEvent` to events, rolls `chance`, applies effect. Examples: lifesteal on damage, revive on death.

3. **Formula-modifying relics** — Checked inline in `DamageSystem.calculateDamage()` and `SkillSystem.applyHeal()`. New "rule-changing" relics that alter combat math. Examples: ignore defense, overheal to shield.

### Lifecycle

- `RelicSystem.activate(heroes, relics)` — called at `BattleScene.create()` after units
- Registers EventBus listeners for reactive relics
- `RelicSystem.deactivate()` — called at `BattleScene.shutdown()`, removes listeners
- `RelicSystem.getStatModifiers(unitId)` — called by `getEffectiveStats()` for passive relics
- `RelicSystem.getFormulaModifiers()` — called by DamageSystem for rule-changers

### File: `src/systems/RelicSystem.ts`

Singleton pattern. Holds reference to current run's relics. Provides static query methods for DamageSystem/SkillSystem to check formula modifiers without tight coupling.

## 8 New Build-Defining Relics

### Rule-changers (4) — `formula_mod` type

| ID | Name | Effect | Integration |
|----|------|--------|-------------|
| `armor_piercer` | 破甲之刃 | 攻击无视目标30%防御 | DamageSystem formula |
| `overflow_shield` | 溢疗护盾 | 治疗溢出转为护盾(最多20%maxHP) | SkillSystem heal pipeline |
| `chain_reaction` | 连锁反应 | 元素反应伤害+50%，溅射30% | ElementSystem reaction |
| `glass_cannon` | 玻璃大炮 | 攻击+60%，受伤+30% | DamageSystem formula (both) |

### Build-definers (4) — conditional `passive` type

| ID | Name | Effect | Integration |
|----|------|--------|-------------|
| `mono_element_crown` | 元素统一王冠 | 全队同元素时伤害+40% | getEffectiveStats() |
| `diversity_badge` | 多元徽章 | 每种不同职业+8%攻速 | getEffectiveStats() |
| `berserker_oath` | 狂战士誓约 | 无治疗者时攻击+20%暴击+10% | getEffectiveStats() |

## Type Changes

Extend `RelicEffect.type` with `'formula_mod'` for rule-changing relics.

## Testing Strategy

- Unit: RelicSystem stat modifiers, event triggers, formula modifications
- Integration: relic + DamageSystem, relic + heal pipeline
- Content integrity: all 43 relics valid structure

## Design Principles

- No changes to existing relic JSON structure (backward compatible)
- Relics that don't apply in battle (display-only) gracefully skip
- `?? defaultValue` guards for old saves without new relic fields
