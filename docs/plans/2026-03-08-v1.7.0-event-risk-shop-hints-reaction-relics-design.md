## Part A: Event Choice Risk Tags

### Current State
EventScene shows probability labels per outcome ("60% 有利 | 40% 无事"). No visual risk indicator per choice.

### Design

Add a colored risk tag to each choice button based on worst-case outcome severity:

- **低风险** (green `#44ff44`) — worst outcome is neutral or minor loss (≤10% max HP damage, or ≤20 gold loss)
- **中风险** (yellow `#ffcc00`) — worst outcome is moderate (>10% HP, >20 gold, or negative stat_boost)
- **高风险** (red `#ff4444`) — worst outcome involves sacrifice, transform, or >30% max HP loss

**Risk calculation**: For each choice, scan all outcomes' effects. Find the worst-case effect. Classify:
- `heal`, `gold` (positive), `item`, `relic`, `recruit` → low risk
- `damage` with value ≤ 10% max HP or `gold` loss ≤ 20 → low risk
- `damage` with value > 10% max HP or `gold` loss > 20 → medium risk
- `sacrifice`, `transform`, or `damage` > 30% max HP → high risk
- `stat_boost` with negative value → medium risk
- If no negative outcomes exist → low risk

Note: max HP reference comes from average party HP. For simplicity, use a fixed threshold: damage ≤ 15 = low, 16-40 = medium, >40 = high.

**Display**: Small text `[低风险]` / `[中风险]` / `[高风险]` to the right of each choice text, colored accordingly. Font size 9px.

- Modify: `src/scenes/EventScene.ts` — add risk tag display next to choice buttons
- No data model changes needed

## Part B: Map Node Info Enhancement

### B1. Shop Tooltip

**Current**: Shop nodes show generic "浏览和购买装备".

**Design**: Show category + price range:
- "武器为主 · 15-45金" (weapon-focused)
- "防具为主 · 20-50金" (armor-focused)
- "饰品为主 · 10-35金" (accessory-focused)
- "综合商店 · 10-55金" (mixed)

**Category logic**: Count items by slot (weapon/armor/accessory). If one slot has >50% of items, that's the category. Otherwise "综合商店".

**Data**: Add optional `shopHint` to ShopNodeData:
```typescript
interface ShopNodeData {
  items: ItemData[];
  shopHint?: { category: string; minPrice: number; maxPrice: number };
}
```

ShopGenerator computes `shopHint` when creating the shop. NodeTooltip reads it for display.

### B2. Event Tooltip Risk Level

Add risk level to map node tooltip for event nodes. Compute the highest risk level among all choices in the event. Display after the event title:
- "神秘商人 [低风险]" (green)
- "暗影祭坛 [高风险]" (red)

This helps route-planning decisions on the map.

### Files
- Modify: `src/types/index.ts` — add shopHint to ShopNodeData
- Modify: `src/systems/ShopGenerator.ts` — compute shopHint when generating shop items
- Modify: `src/ui/NodeTooltip.ts` — display shop category/price + event risk level

## Part C: Reaction-Specific Trigger Relics

### Current State
3 existing element reaction relics all boost raw reaction damage (elemental_catalyst +25%, elemental_fusion_stone +35%, chain_reaction +50% + splash). No relics trigger unique effects on specific reaction types.

### Design — 5 New Relics

| ID | Name | Reaction | Rarity | Effect |
|----|------|----------|--------|--------|
| melt_heart | 融化之心 | Melt (fire+ice) | rare | Heal attacker for 20% of reaction damage |
| overload_engine | 超载引擎 | Overload (fire+lightning) | rare | 25% chance to stun target for 1s |
| superconduct_shield | 超导护盾 | Superconduct (ice+lightning) | rare | Grant attacker 10 temp shield on proc |
| annihilation_echo | 湮灭回响 | Annihilation (dark+holy) | legendary | Repeat 40% of reaction damage after 1s delay |
| elemental_resonance | 元素共鸣 | Any reaction | legendary | Reduce target element resistance by 15% for 5s |

### Trigger Mechanism

RelicSystem already subscribes to `element:reaction` event. The event data includes:
- `reactionType: string` — "melt", "overload", "superconduct", "annihilation"
- `attacker: Unit` — the unit that triggered the reaction
- `target: Unit` — the unit that received the reaction
- `damage: number` — reaction damage dealt

**Verify**: Check that `element:reaction` event includes `reactionType` in its payload. If not, add it from ElementSystem.

### Relic Data Extension

Add `reactionType` filter field to relic config:
```typescript
// In relics.json, new entries:
{
  id: "melt_heart",
  triggerEvent: "element:reaction",
  effect: {
    type: "on_reaction",
    reactionType: "melt",     // NEW: filter field
    healPercent: 0.2           // custom param
  }
}
```

### RelicSystem Integration

In RelicSystem's `element:reaction` handler, after existing damage bonus calculation:
1. Check if any held relics have `reactionType` matching the event's reaction type (or "any")
2. For each matching relic, apply its unique effect:
   - `melt_heart`: `attacker.heal(damage * 0.2)` via DamageSystem.applyHeal
   - `overload_engine`: `rng.chance(0.25)` → apply stun status effect to target
   - `superconduct_shield`: add 10 to attacker's shield via HealthBar
   - `annihilation_echo`: `scene.time.delayedCall(1000, () => target.takeDamage(damage * 0.4))`
   - `elemental_resonance`: apply element_resist_down status effect to target (15%, 5s)

## Design Principles

1. No new systems — extends existing EventScene, NodeTooltip, ShopGenerator, RelicSystem
2. Risk calculation is pure display logic — no gameplay impact
3. Shop hints computed at generation time, not on-the-fly
4. Reaction relics use existing EventBus pattern — no new subscription types
5. Backward-compatible — shopHint uses `??` guard for old saves
