# Formation System Design

> 日期: 2026-03-09
> 状态: 已批准
> 目标: 添加前后排阵型系统，为战斗增加站位策略决策

## 概述

前后排系统：玩家将每个英雄分配到前排或后排。分配在整个 run 中持续有效，可随时在地图界面修改。排位影响**敌人选择目标的优先级**，不影响属性数值。

## 数据模型

HeroState 新增一个字段：

```typescript
export interface HeroState {
  // ... existing fields
  formation: 'front' | 'back';
}
```

**自动分配默认值**（选择英雄/招募时）：
- `tank`, `melee_dps` → `'front'`
- `ranged_dps`, `healer`, `support` → `'back'`

## 战斗站位

将单列布局替换为两个 X 位置：

| 排位 | X 位置 | 说明 |
|------|--------|------|
| 前排 | ~140 | 更靠近敌人，吸引近战仇恨 |
| 后排 | ~80 | 远离敌人，更安全 |

同排内英雄按现有方式垂直居中（UNIT_SPACING_Y 间距）。2-3 个英雄可产生以下布局：
- 1前+2后、2前+1后、1前+1后 等

## 目标选择影响

TargetingSystem 变更 — 敌方近战单位选择目标时：
- 前排英雄获得**目标权重加成**（如 +50 分）
- 仅适用于近战敌人（远程敌人按正常威胁/距离选择目标）
- 若所有前排英雄死亡，后排正常成为目标

在评分函数中添加一行：`if (candidate.formation === 'front') score += FRONT_ROW_AGGRO_BONUS`。

## UI：地图界面阵型按钮

地图界面上的小按钮（"阵型"），打开一个模态面板：
- 左区域："前排" — 放置区
- 右区域："后排" — 放置区
- 英雄 chibi 精灵可在两列之间**拖拽**（或点击切换）
- 自动分配按钮重置为角色默认值

使用现有 **Panel + 模态模式**（背景 d799，面板 d800，关闭按钮 d801）。

## 场景流程（不变）

```
HeroDraftScene → (自动分配阵型) → MapScene → [可选：打开阵型面板] → BattleScene
```

无新场景，无战前中断。

## 存档兼容

HeroState 上的 `formation` 字段使用 `?? autoAssignByRole(heroData.role)` 保护旧存档。

## 测试

- 单元测试：按角色自动分配
- 单元测试：TargetingSystem 前排加成评分
- 单元测试：阵型通过存档/读档持续
- 集成测试：战斗位置使用阵型排位

## 设计原则

1. **仅目标优先级** — 不修改属性，保持 effective-stats 管线简洁
2. **持久但可调** — 默认按角色分配，玩家可随时在地图界面修改
3. **向后兼容** — 旧存档缺失字段时自动回退到角色默认值
4. **YAGNI** — 不添加阵型 buff、不添加多列网格、不添加预设模板
